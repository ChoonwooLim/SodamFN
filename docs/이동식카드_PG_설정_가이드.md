# 📱 이동식 카드매출 (코페이·KSnet 등) 설정 가이드

> **이동식 단말기 카드매출**(POS 미경유)을 자동으로 매출에 잡고 수수료를 정확히 계산하는 방법

---

## 🤔 이게 뭔가요?

가게에서 카드결재는 보통 **계산대의 POS 단말기**(이지페이 등)로 받습니다. 이건 매장 매출에 자동으로 잡혀요.

그런데 가끔 **이동식 카드 단말기**(테이블 앞에서 결재, 야외/이벤트 결재, 대용량 주문 응대 등)를 쓰시면 다릅니다:

- POS에 안 잡힘 → 매출 데이터에 누락 위험
- 카드사가 직접 입금하는 게 아니라 **중개 회사(코페이·KSnet 등) 거쳐서** 수수료 떼고 입금
- 예: **결재 200,000원 → 수수료 5,500원 (2.75%) 차감 → 통장에 194,500원 입금**

셈하나 시스템은 이런 입금을 자동으로 인식해서:
1. ✅ **실제 매출 200,000원으로 정확히 기록**
2. ✅ **수수료 5,500원을 별도로 추적**
3. ✅ **손익관리·세무 신고에 정확히 반영**

---

## 🚀 처음 설정하기 (1분이면 됩니다)

### 1단계: 정산·수수료 탭으로 이동
```
손익관리 → 은행계좌 연동 → 정산·수수료 탭
```

### 2단계: [📱 이동식 PG 설정] 버튼 클릭
화면 우측 상단의 하늘색 버튼입니다.

### 3단계: [+ 새 PG 추가]
모달이 열리면 점선 박스 [+ 새 PG 추가] 클릭.

### 4단계: 4가지 항목 입력

| 항목 | 입력 예시 | 설명 |
|------|----------|------|
| **표시명** | `코페이` | 시스템에 표시될 이름 (자유) |
| **적요 키워드** | `코페이` | 통장 적요에 들어가는 글자 (대소문자 구분) |
| **수수료율** | `2.75` | 정산 명세서에 적힌 수수료율 (% 단위) |
| **메모** | `2026년 5월 명세 기준` | 나중에 참고하실 메모 (선택) |

### 5단계: [저장]
끝입니다. 다음에 코페이 입금이 들어오면 자동으로 매출 200,000원 + 수수료 5,500원으로 처리됩니다.

---

## 💡 자주 쓰시는 PG들 — 입력 예시

### 코페이
```
표시명: 코페이
키워드: 코페이
수수료율: 2.75
```

### KSnet
```
표시명: KSnet
키워드: KSNET    (대소문자 정확히 — 통장에 어떻게 찍히는지 확인하세요)
수수료율: 3.00
```

### 키움페이
```
표시명: 키움페이
키워드: 키움
수수료율: 2.85
```

### 한국정보통신 (KICC)
```
표시명: 한국정보통신
키워드: KICC    (또는 한국정보)
수수료율: 2.90
```

> **수수료율은 반드시 실제 정산 명세서를 보고 입력하세요.** 위 숫자는 일반적인 예시일 뿐 사장님 계약에 따라 다를 수 있습니다.

## 📊 과거 거래도 자동 정리하고 싶어요

PG를 등록만 하면 **앞으로 들어올** 입금이 자동 분류됩니다. 그런데 **이미 통장에 들어왔던 과거 코페이 입금**들은 그대로 남아 있어요.

이걸 한 번에 정리하려면:

### 1단계: 거래내역 탭으로 이동
정산·수수료 탭 옆의 **거래내역** 탭 클릭.

### 2단계: [정산 재분류] 버튼 클릭 (보라색)
필터 적용 옆의 보라색 [정산 재분류] 버튼.

### 3단계: 확인
"카드사·페이·배달앱·이동식 키워드 매칭되는 입금을 재분류합니다" 안내 → [확인]

→ 코페이/KSnet 등 등록된 PG에 매칭되는 모든 과거 입금이 한꺼번에 **이동식카드** 분류로 바뀌고 매출에 자동 반영됩니다.

> ⚠️ 사장님이 **수동으로 분류해두신 거래는 절대 건드리지 않습니다.** 안전합니다.

---

## ✏️ 나중에 수수료율이 바뀌면

### 코페이가 수수료율을 인상한 경우 (예: 2.75% → 3.0%)

```
1. [📱 이동식 PG 설정] 다시 클릭
2. 코페이 항목 옆 [수정] 클릭
3. 수수료율 2.75 → 3.0 변경
4. [저장]
5. 거래내역 탭 → [정산 재분류] → 과거 거래도 새 수수료율로 자동 재계산
```

> 새 수수료율은 **변경 시점 이후 입금**부터 적용됩니다. 과거 거래 재계산이 필요하면 [정산 재분류]를 누르시면 됩니다.

---

## 🛠️ 일시 중단하고 싶어요 (계약 해지 등)

PG를 삭제하지 않고 잠시 **비활성**으로 두실 수 있어요:

```
[📱 이동식 PG 설정] → 해당 PG 옆 [활성] 토글 클릭
→ 회색 [비활성]으로 변경됨
→ 앞으로 들어오는 입금에는 이 룰이 적용되지 않음
→ 과거 분류된 거래는 그대로 유지
→ 나중에 다시 쓰시면 [비활성] 토글로 활성화
```

완전 삭제는 휴지통 아이콘 클릭.

## ❓ 자주 묻는 질문

### Q1. 키워드를 어떻게 정해야 하나요?
**통장 거래내역의 적요(보내신 분)**에 들어 있는 글자를 그대로 입력하세요.

예: 화면에 `코페이 (기업)` 으로 보이면 → 키워드는 `코페이`
- 너무 짧으면 안 됩니다 (예: `코` 만 쓰면 다른 입금도 잘못 잡힐 수 있어요)
- 너무 길어도 안 됩니다 (예: `코페이 (기업)` 까지 정확히 안 들어 있으면 안 매칭)
- **3~5글자 정도, 다른 입금에 안 들어가는 고유한 단어**가 좋습니다

### Q2. 수수료율을 모르겠어요
PG 회사에서 **월말 정산 명세서**를 보내드립니다. 거기에 수수료율이 명시되어 있어요.

명세서를 못 받으셨다면:
- 일단 **2.75% 대략 입력** → 한 달 운영 후 명세 받으시면 정확히 조정

또는:
- 영수증 종이에 작게 적혀 있는 경우도 있으니 확인

### Q3. 한 가게에 코페이랑 KSnet 둘 다 쓰는데요?
**둘 다 등록하세요.** 각각 다른 키워드·수수료율로 등록 가능합니다.

### Q4. 매장이 여러 개인데 매장마다 다른 PG 써요
**각 매장 관리자가 자기 매장에서 등록**하시면 됩니다. 매장별로 독립적으로 관리됩니다.

### Q5. 수동으로 분류해 둔 거래도 자동으로 바뀌나요?
**아니요.** 사장님이 수동으로 분류하신 거래는 절대 자동으로 안 바뀝니다. 안전합니다.

만약 수동 분류한 거래도 PG 룰로 다시 바꾸고 싶으시면:
- 해당 거래의 분류 드롭다운에서 **이동식카드**로 직접 변경

### Q6. 분류가 잘못 됐어요. 어떻게 수정하나요?
거래내역 탭에서 해당 거래 행의 **분류 드롭다운을 직접 변경**하시면 됩니다. 수동 분류는 시스템이 학습해서 다음에 비슷한 거래가 들어오면 같은 분류로 자동 적용합니다.

### Q7. 손익관리에는 어떻게 반영되나요?

| 항목 | 처리 |
|------|------|
| **매출** | 결재 원본 200,000원이 매출에 포함 (자동) |
| **수수료** | 5,500원이 별도 추적 (정산·수수료 탭에서 확인) |
| **실입금** | 통장 입금 194,500원 (거래내역 그대로) |

세무사님께 자료 보내실 때 매출은 200,000원으로 정확히 신고됩니다.

## ⚠️ 주의사항

1. **수수료율 직접 확인 필수** — 시스템이 자동 계산하지만 PG사가 임의로 바꾸기도 합니다. 분기마다 명세서 보고 조정해주세요.
2. **키워드 신중히 입력** — 너무 짧으면 다른 입금이 잘못 잡힐 수 있어요. 등록 후 [정산 재분류] 결과 보고 이상 있으면 키워드 조정.
3. **VAT(부가세) 처리** — 이 가이드의 수수료율은 부가세 포함 여부에 따라 다를 수 있습니다. 세무사님과 협의 후 정확한 값 입력.
4. **여러 PG가 같은 키워드 쓰면** — 시스템이 첫 매칭만 적용합니다. 가급적 PG마다 명확히 구분되는 키워드 사용.
